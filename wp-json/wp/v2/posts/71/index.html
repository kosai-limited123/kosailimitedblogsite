What is CDT?<\/h3>\n\n\n\n
<\/div>\n\n\n\n

In CDT the data representation is known. And a user can change the representation too. And all these based on operation are to be done. It’s the opposite of ADT<\/a>. It can be embedded or composed with other data types to form larger data types. <\/p>\n\n\n\n

<\/div>\n\n\n\n

Example<\/h3>\n\n\n\n
<\/div>\n\n\n\n

A parse tree<\/a> for Haskell is a concrete data type. Suppose a set of chair was brought and arranged in a way for the people to sit. Now it depends on us how to rearrange the chairs , and whom we should allow to seat. And you can also define for which purpose the chair will be used. This defines CDT.<\/p>\n\n\n\n
